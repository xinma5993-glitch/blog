Git、GitHub 与第一次提交

2026-07-14 Git GitHub

第一次把项目推到 GitHub 时，最容易混淆的是三个概念：本地仓库、提交和远程仓库。本地仓库保存项目历史，提交是某一刻的快照，远程仓库则负责备份和协作。

一个稳妥的流程通常是先确认当前目录内容，再创建或检查 `.gitignore`，然后设置提交作者、创建提交、配置 `origin`，最后推送到远程分支。每一步都可以通过 `git status` 看清楚当前状态。

为什么先提交再推送

GitHub 只能接收已经存在于本地历史中的提交。如果工作区里只是有文件，但没有 `git commit`，推送时不会把这些文件带到远程。把提交和推送分开，也方便在上传前做敏感信息检查。

HTTPS 与 SSH

HTTPS 适合配合 GitHub CLI 使用，登录后 Git 可以通过凭据管理器完成认证。SSH 则需要本机有公钥并把公钥添加到 GitHub。两种方式都可以，关键是 remote 地址要和认证方式一致。
